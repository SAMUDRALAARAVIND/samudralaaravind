2D arrays Vs Pointers:

In the previous chapter we learnt that the name of a 1D array is itself the address of first element of that array.What will be the answer in case of 2D array,3D array...Multi dimensional array??. you will find the answer for this in this chapter.

before getting into this chapter lets analyse anexample below

#include<iostream>
using namespace std;
int main()
{
int a[5]={23,44,32,67,-78};/*declaring a container of memory 20 bytes/*
cout<<a<<" and "<<a+1<<endl;
cout<<&a<<" and "<<&a+1<<endl;
cout<<*&a;
return 0;
}



output:
0x6ffe10 and 0x6ffe14
0x6ffe10 and 0x6ffe1c
0x6ffe10


0x6ffe14-0x6ffe10=4
0x6ffe1c- 0x6ffe10=20

Explanation of above code is as follows:

Explanation:

In the above code it's very clear that the name of the array a is pointing to an integer at 0th index. that's why when we add 1 to a we are shifted by size of int(4 bytes),whereas &a is ponting to the whole array that's why when we add 1 to &a we are shifted by size of the array(20 bytes).

The third line in our output tells that dereferecing a ponter to array will give us pointer to integer of first index.

Pointer to 2-D array:

In a two dimensional array, we can access each element by using two indices, where first index represents the row number and second index represents the column number. The elements of 2-D array can be accessed with the help of pointer notation also. Suppose arr is a 2-D array, we can access any element arr[i][j] of the array using the pointer expression *(*(arr + i) + j). Now weâ€™ll see how this expression can be derived. Let us take a two dimensional array a[3][4]:

int a[3][4]={ {33,44,23,-29} , {45,86,64,35}, {79,103,208,100}};

Since memory in a computer is organized linearly it is not possible to store the 2-D array in rows and columns. The concept of rows and columns is only theoretical, actually, a 2-D array is stored in row-major order i.e rows are placed next to each other.
